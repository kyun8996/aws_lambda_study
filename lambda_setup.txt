# 2. 함수 이름 설정, 런타임은 Python 3.x.x (로컬 실행 환경과 동일하게 설정), 아키텍처는 x86으로 설정

# 3. 기본 Lambda 권한을 가진 새 역할(Role) 생성 (미리 IAM에서 생성해도 됨)
# 기본적으로 Lambda는 Amazon CloudWatch Logs에 로그를 업로드할 수 있는 기본 실행 역할을 자동으로 생성합니다.

# IAM 콘솔에 접속하면, Lambda 함수 이름과 동일한 역할이 생성되어 있습니다.
# 역할을 클릭한 뒤 ‘권한 추가 → 정책 연결’을 선택합니다.
# ‘AmazonS3FullAccess’, ‘AWSLambdaBasicExecutionRole’(기본 정책을 실수로 삭제했을 경우 다시 추가)
# 이 두 정책을 연결하면 Lambda 함수가 S3에 접근할 수 있고, 로그를 CloudWatch에 기록할 수 있습니다.

# 4. CloudWatch 로그 확인 방법
# CloudWatch 콘솔 → 로그 그룹(Log groups) → Lambda 함수 이름 선택 → 로그 스트림(Log stream) 확인
